# Script YAML Schema

This document describes the screenplay YAML generated by Novel2Script Agent Studio.

The single source of truth is the Pydantic v2 model in `backend/app/domain/schemas.py`. The backend generates `schema.json` from that model and uses the same schema for backend validation and frontend editor validation.

## Design Goals

- **Editable by authors**: YAML keeps the draft readable and easy to revise.
- **Structured for tools**: characters, locations, scenes, actions, and dialogue are explicit objects.
- **Traceable to chapters**: each scene records the source chapter indexes it adapts.
- **Safe for AI output**: the LLM generates structured JSON first; the program validates it and exports YAML.

## Top-level Fields

| Field | Type | Purpose |
|---|---|---|
| `metadata` | object | Script title, source chapter count, language, genre, and logline |
| `characters` | array | Canonical character list used by scenes and dialogue |
| `locations` | array | Reusable locations referenced by scenes |
| `props` | array | Important props or objects |
| `scenes` | array | Ordered screenplay scenes |
| `adaptation_notes` | array | Notes about compression, omissions, tone, and future polishing |

## Scene Fields

Each scene includes:

- `id`: stable scene id, such as `sc_001`.
- `title`: short scene title.
- `source_chapters`: chapter indexes adapted by the scene.
- `location_id`: id from `locations`.
- `time_of_day`: broad time label.
- `characters`: character ids appearing in the scene.
- `purpose`: dramatic purpose of the scene.
- `actions`: stage/action cues.
- `dialogues`: spoken lines.
- `adaptation_notes`: scene-level adaptation notes.

## Validation Rules

The backend checks:

- YAML is parseable.
- Required schema fields are present.
- Scene ids are unique.
- Scene characters reference existing character ids.
- Dialogue speakers reference existing character ids.
- Scene locations reference existing location ids.
- Each scene has at least one action or one dialogue line.
- At least three source chapters are represented in the generated scenes.

## Example

```yaml
metadata:
  title: 雨巷里的来信
  source_chapter_count: 3
  language: zh-CN
  genre: drama
  logline: A young archivist follows three letters into a hidden family secret.
characters:
  - id: char_protagonist
    name: 林夏
    role: protagonist
    description: A careful young archivist.
    first_appearance_chapter: 1
locations:
  - id: loc_archive
    name: 城南档案馆
    description: An old archive building near the rain alley.
props: []
scenes:
  - id: sc_001
    title: 第一封信
    source_chapters: [1]
    location_id: loc_archive
    time_of_day: evening
    characters: [char_protagonist]
    purpose: Establish the mystery.
    actions:
      - text: 林夏在旧档案盒中发现一封没有署名的信。
        beat: discovery
    dialogues:
      - speaker_id: char_protagonist
        line: 这不是馆里的编号。
        emotion: confused
    adaptation_notes:
      - Condense chapter exposition into a visual discovery.
adaptation_notes:
  - Keep chapter-to-scene mapping explicit for later editing.
```
