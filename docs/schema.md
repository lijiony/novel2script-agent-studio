# Script YAML Schema

This document defines the screenplay YAML generated by Novel2Script Agent Studio.

The single source of truth is the Pydantic v2 model in `backend/app/domain/schemas.py`. The backend generates `schema.json` from that model and uses the same schema for backend validation and frontend editor validation.

## Design Goals

- **Adaptation co-writer, not formatter**: YAML stores the draft plus why the AI adapted it this way.
- **Author control**: the selected script format, adaptation scale, style focus, preserved items, forbidden changes, and author notes are saved in the final YAML.
- **Traceable scenes**: each scene records source chapters and source excerpts.
- **Performable writing**: scenes include purpose, conflict, emotional shift, action, dialogue, and production risk.
- **Revision-ready**: each scene can include AI-added content and revision suggestions.
- **Safe AI output**: the LLM generates structured JSON first; the program validates it and exports YAML.

## Top-level Fields

| Field | Purpose |
|---|---|
| `metadata` | Script title, source chapter count, language, genre, and logline |
| `adaptation_profile` | Author choices: format, style, scale, preserved items, forbidden changes, notes |
| `adaptation_strategy` | Why this adaptation direction was chosen |
| `characters` | Canonical character list used by scenes and dialogue |
| `locations` | Reusable locations referenced by scenes |
| `props` | Important props or objects |
| `scenes` | Ordered screenplay scenes with source tracking and revision metadata |
| `adaptation_notes` | Project-level notes about compression, omissions, tone, and future polishing |

## Author Control Fields

`adaptation_profile` records the author's generation brief:

- `format_type`: `film`, `short_drama`, `stage_play`, `radio_drama`, `animation`, or `game_script`.
- `adaptation_scale`: `faithful`, `balanced`, or `bold`.
- `style_focus`: `psychological`, `action`, `dialogue`, `suspense`, `relationship`, or `custom`.
- `preserve_items`: items the AI should keep.
- `forbidden_changes`: items the AI should not change.
- `author_notes`: free-form author instruction.

## Scene Fields

Each scene includes:

- `id`: stable scene id, such as `sc_001`.
- `title`: short scene title.
- `source_chapters`: source chapter indexes.
- `source_excerpt`: source prose used by the adaptation.
- `location_id`: id from `locations`.
- `time_of_day`: broad time label.
- `characters`: character ids appearing in the scene.
- `purpose` and `scene_purpose`: dramatic function of the scene.
- `conflict`: what pushes the scene dramatically.
- `emotional_shift`: how the emotional state changes.
- `production_risk`: what might make the scene hard to perform or shoot.
- `format_type`: script format used by this scene.
- `actions`: playable action cues.
- `dialogues`: spoken lines.
- `ai_added_content`: content added by AI beyond the direct source.
- `revision_suggestions`: concrete next edits for the author.
- `adaptation_notes`: scene-level adaptation explanation.

`actions` and `dialogues` include `origin`:

- `source_extracted`: copied or closely extracted from the source.
- `ai_adapted`: adapted from source meaning into performable script.
- `ai_added`: newly added by AI to improve performance, conflict, or clarity.

## Validation Rules

The backend checks:

- YAML is parseable.
- Required schema fields are present.
- Scene ids are unique.
- Scene characters reference existing character ids.
- Dialogue speakers reference existing character ids.
- Scene locations reference existing location ids.
- Each scene has at least one action or one dialogue line.
- At least three source chapters are represented in the generated scenes.

## Example

```yaml
metadata:
  title: 雨巷里的来信
  source_chapter_count: 3
  language: zh-CN
  genre: mystery drama
  logline: A young archivist follows hidden letters into an abandoned theater.
adaptation_profile:
  format_type: short_drama
  adaptation_scale: balanced
  style_focus: psychological
  preserve_items:
    - 保留林夏和周砚的关系
  forbidden_changes:
    - 不要改变父亲失踪设定
  author_notes: 心理活动尽量转成动作。
adaptation_strategy:
  - Track source chapters and excerpts so the author can audit what changed.
characters:
  - id: char_linxia
    name: 林夏
    role: protagonist
    description: A careful young archivist.
    first_appearance_chapter: 1
locations:
  - id: loc_archive
    name: 城南档案馆
    description: An old archive building near the rain alley.
props: []
scenes:
  - id: sc_001
    title: 第一幕 雨巷里的信
    source_chapters: [1]
    source_excerpt: 林夏在城南档案馆值夜班，发现没有编号的信。
    location_id: loc_archive
    time_of_day: night
    characters: [char_linxia]
    purpose: Establish the mystery.
    scene_purpose: Turn inner suspicion into a playable discovery.
    conflict: 林夏想查明信件来源，但线索指向父亲失踪。
    emotional_shift: From guarded curiosity to active determination.
    production_risk: Avoid explaining the clue only through narration.
    format_type: short_drama
    actions:
      - text: 林夏把信举到灯下，停顿，重新看父亲留下的旧档案。
        beat: discovery
        origin: ai_adapted
    dialogues:
      - speaker_id: char_linxia
        line: 这不是馆里的编号。
        emotion: confused
        origin: ai_adapted
    ai_added_content:
      - Add a silent hesitation beat to externalize doubt.
    revision_suggestions:
      - Add a stronger visible obstacle if the opening feels too quiet.
    adaptation_notes:
      - Condense prose exposition into action and concise dialogue.
adaptation_notes:
  - Keep chapter-to-scene mapping explicit for later editing.
```
