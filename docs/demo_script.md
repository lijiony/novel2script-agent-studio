# 90-second Demo Script

0-10s:
大家好，这是我的 XEngineer 第三批次作品 Novel2Script Agent Studio，选题是 AI 小说转剧本工具。它不是单纯把小说转成 YAML，而是一个 AI 改编副编剧。

10-25s:
这里我加载一份三章小说示例。系统先创建一次性的 run_id，不使用数据库，并执行章节检测、人物地点事件抽取和改编计划生成。

25-42s:
点击“分析小说”后，中间栏会出现 AI 改编计划：推荐剧本类型、风格方向、场景规划、人物线索和潜在风险。这个阶段类似计划模式，先解释怎么改，再让作者确认。

42-58s:
作者可以选择短剧、影视、舞台剧等格式，也可以选择忠实版、平衡版或大胆改编，并填写必须保留和禁止改动的内容。这里我选择短剧、忠实版、心理外化。

58-73s:
点击“生成剧本”后，后端根据计划和作者控制项生成结构化 JSON，再用 Pydantic 和 JSON Schema 校验，最后由程序用 `ruamel.yaml` 导出 YAML，LLM 不直接输出 YAML。

73-84s:
生成结果里每场戏都有场景功能、冲突、情绪变化、原文片段、AI 新增内容、修改建议和生产风险。作者可以继续编辑 YAML，并通过后端重新校验。

84-90s:
右侧可以下载计划、YAML、Schema 和改编报告。这个 MVP 展示的是从小说理解、作者控制到可追踪剧本初稿的完整闭环。
